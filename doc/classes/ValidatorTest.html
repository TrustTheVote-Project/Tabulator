Class ValidatorTest
In: test/validator_test.rb
Parent: Test::Unit::TestCase

The ValidatorTest class provides Unit Testing for the Tabulator Validator. It tests for all possible error and warning messages generated by the Tabulator during its data validation process.

tabulator_messages_generate(messages, header, printit = false)

Arguments:

  • messages: (Array) of error/warning messages
  • header: (Array) of error/warning messages
  • printit: (Boolean) whether to print the error/warning messages (optional)

Returns: String

Returns a String containing the concatenation of all of the error/warning messages, each preceded by the header and followed by a newline character. If printit is true, the messages are printed before being returned.

tabulator_messages_unsame(received, expected, label, header)

Arguments:

  • received: (Array) of received error messages
  • expected: (Array) of expected warning messages
  • label: (String) either "Errors" or "Warnings"
  • header: (String) either ERRHEAD or WARHEAD

Returns: String

Called only when the sets of received and expected messages are not of the same size. Collects into and returns a single message String consisting of the expected error/warning messages followed by those actually received.

tabulator_print_errors_warnings(tab)

Arguments:

  • tab: (Tabulator) Tabulator object

Returns: N/A

Prints all error and warning messages currently held by the Tabulator.

validator_test_add_cc(trace, dir, cc_file, errors, warnings)

Arguments:

  • trace: (Integer) limits tracing of output for the syntax checker
  • cc_file: (String) name of file holding Counter Count
  • errors: (Array) of expected error messages
  • warnings: (Array) of expected warning messages

Returns: N/A

Tests the accumulation of a new Counter Count after instantiating the Tabulator. The proper number of errors and warnings should appear, and they should match exactly, content-wise.

validator_test_check_syntax(trace, prefix, dir, file)

Arguments:

  • trace: (Integer) limits tracing of output for the syntax checker
  • prefix: (String) prefix of the schema file name
  • file: (String) Tabulator data file name

Returns: Hash

Read a schema from one file (under data/Schemas/) and a datum from another file from dir/, and then tests the success of performing a syntax check of the datum against the schema.
